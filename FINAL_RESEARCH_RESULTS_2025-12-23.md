## Results by Hypothesis

### H1: Index-Based Generator
**Theory**: drift[k][lane] = f(k, lane)

**Tests Run**:
1. **Statistical correlation**: Lanes 2-7 show moderate correlation (0.53-0.69)
2. **Polynomial fits**: 46.65% overall (failed for complex lanes)
3. **Modular arithmetic**: **69.57% overall** ‚Üê BEST
4. **PySR symbolic regression**: 12.50% (failed - discovered irrelevant formula)

**Key Findings**:
- **Lanes 9-15**: 100% match with `drift = (0*k + 0) mod 256` = **drift = 0**
- **Lane 8**: 91.3% match with drift = 0
- **Lanes 0-7**: Failed (<8% accuracy) - complex patterns

**Conclusion**: Index-based works ONLY for lanes 9-15 (drift=0), fails for active lanes

---

### H2: Cryptographic Hash Functions
**Theory**: drift[k][lane] = hash(k, lane) mod 256

**Tests Run**:
1. SHA256, SHA512, MD5
2. Bitcoin-specific: double SHA256, HASH160 (RIPEMD160)
3. Various encodings: (k, lane), (k||lane), str concat, byte concat
4. Salt variations

**Best Result**: SHA512 with string concatenation - **0.8%**

**Conclusion**: ‚ùå **FAILED** - Drift is NOT generated by cryptographic hashes

**Insight**: If it were a hash, we'd see 100% or 0%, not 0.8%. This rules out hash-based generation.

---

### H3: PRNG (Pseudo-Random Number Generator)
**Theory**: rng = PRNG(seed); drift = rng.next() mod 256

**Tests Run**:
1. Python random.Random() - best seed: 12345 (0.9%)
2. NumPy random - best seed: 0 (0.7%)
3. **LCG (Linear Congruential)** - best: **MINSTD seed=0 (69.2%)**
4. Brute force seed search (0-100,000) - best: seed 15425 (15.0%)
5. Custom PRNG patterns - failed

**Best Result**: LCG (MINSTD variant) - **69.2%**

**Conclusion**: ‚ùå **FAILED** - Drift is NOT pure PRNG output

**Insight**: 69.2% with LCG suggests some mathematical structure, but not reproducible PRNG sequence

---

### H4: Recursive Pattern (Drift Ladder)
**Theory**: drift[k+1] = f(drift[k])

**Tests Run**:
1. **Affine recurrence**: drift[k+1] = (A*drift[k] + C) mod 256 - **70.5% overall** ‚Üê BEST
2. Polynomial recurrence: drift[k+1] = drift[k]^n + C - 69.3%
3. **Bridge spacing pattern**: drift[k] = drift[k-5] - **67.6% overall, 100% for lanes 9-15!**
4. Multi-step recurrence: Fibonacci-like - 69.5%

**MAJOR DISCOVERY** - **Lanes 9-15 (7 lanes)**:
```
drift[k] = drift[k-5] + 0

100% MATCH FOR ALL 7 LANES!
```

**Per-Lane Breakdown**:
- **Lanes 0-6**: Complex (20-59 unique drift values)
- **Lane 7**: 82.4% (affine: A=23, C=0)
- **Lane 8**: 92.6% (affine: A=1, C=0)

## ADDENDUM: Activation Phase Discovery (2025-12-23)

### Critical Update to "drift=0" Finding

After attempting to generate puzzles 71-95 using the drift=0 hypothesis, we discovered an important limitation:

**The Data Constraint**:
- Drift data available: Puzzles 1-70 (single-step transitions)
- Actual keys available: Puzzles 1-130 (CSV data)
- Bridge data available: Puzzles 75, 80, 85, 90, 95 (5-step transitions)
- ‚ùå **Missing**: Single-step drift data for puzzles 71-95

**Initial Generation Attempt**:
```bash
python3 generate_lanes_9_15_puzzles_71_95.py
Result: 74.3% accuracy (expected 100%)
```

**Why it Failed**:
The "drift=0 for lanes 9-15" finding was based on puzzles 1-70, where those lanes were **DORMANT**. We attempted to generate puzzles 71-95 assuming drift=0 would continue, but:

1. **Lane activation happens progressively**:
   - Lane 9 activates at k=72 (between bridges 70 and 75)
   - Lane 10 activates at k=80 (exactly at bridge 80)
   - Lane 11 activates at k=88 (between bridges 85 and 90)
   - Lanes 12-15 activate at k=96, 104, 112, 120 (all beyond puzzle 95)

2. **Activation phase analysis** (from bridge data):
   ```
   Bridge 70‚Üí75: Lane 9 goes from 0 ‚Üí 4 (activated with non-zero value!)
   Bridge 75‚Üí80: Lane 9 evolves 4 ‚Üí 234 (varying drift)
   Bridge 80‚Üí85: Lane 10 goes from 0 ‚Üí 17 (activated!)
   Bridge 85‚Üí90: Lane 11 goes from 0 ‚Üí 2 (activated!)
   ```

3. **Multi-step vs Single-step drift**:
   - Bridge transitions are 5-step intervals (k‚Üík+5)
   - Cannot extract single-step drift from 5-step transitions
   - Activations happen MID-BRIDGE, not at bridge endpoints

### Lesson Learned

> **"Dormant phase drift ‚â† Active phase drift"**

The drift=0 finding is valid for lanes in their dormant phase, but once lanes activate, they may exhibit different drift patterns. Our training data (puzzles 1-70) captured mostly dormant lanes, which is why we saw drift=0.

1. Single-step drift data for puzzles 71-95 (currently missing), OR
2. Formula for activation values (how lanes transition from dormant to active), OR
3. Accept we can only generate lanes that remain dormant throughout the target range
